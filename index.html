One-time setup

Complete these steps once. After that, your WHOOP tennis sessions appear in the app automatically every morning — no CSV exports, no manual imports.

1
Get your initial refresh token via Postman
Download Postman (free at postman.com). Use it to complete the WHOOP OAuth flow once and capture your refresh token. Full instructions are in the right panel.
2
Add three secrets to your GitHub repo
Go to your repo on GitHub → Settings → Secrets and variables → Actions → New repository secret. Add these three:
WHOOP_CLIENT_ID → your Client ID WHOOP_CLIENT_SECRET → your Client Secret WHOOP_REFRESH_TOKEN → token from Postman
3
Upload the workflow file to your repo
In your GitHub repo, create the folder path .github/workflows/ and upload sync-whoop.yml into it. GitHub will detect it automatically.
4
Run the workflow manually once to test
Go to your repo → Actions tab → Sync WHOOP Tennis Data → Run workflow. It should complete in under 30 seconds and commit a data.json file to your repo.
5
Done — runs automatically every morning
GitHub Actions will sync at 6am ET daily. New tennis sessions appear in your app automatically. You only need to tag Singles vs Doubles for new sessions.
Getting your refresh token via Postman

Postman has a built-in OAuth flow that handles the browser redirect for you — no server needed for this one-time step.

1
Open Postman → New Collection → Authorization tab → Type: OAuth 2.0
2
Fill in:
Auth URL: https://api.prod.whoop.com/oauth/oauth2/auth
Token URL: https://api.prod.whoop.com/oauth/oauth2/token
Client ID: your Client ID
Client Secret: your Client Secret
Scope: read:workout offline
Redirect URI: https://oauth.pstmn.io/v1/callback
3
Click Get New Access Token → log into WHOOP → authorize. Postman will show you the full token response.
4
Copy the refresh_token value from the response. That goes into GitHub Secrets as WHOOP_REFRESH_TOKEN.
Note on refresh token rotation
WHOOP rotates refresh tokens on each use. The GitHub Action handles this automatically — if the token rotates, a new-refresh-token.txt file appears in your repo. Copy that value and update the WHOOP_REFRESH_TOKEN secret. This rarely happens but worth knowing.
